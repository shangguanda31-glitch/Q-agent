# 修复 schedule_update 更新错日程 + 上下文关联

**日期**: 2026-05-16
**类型**: Bug 修复

## 变更内容
- 修复 schedule_update 按标题匹配脆弱的问题：新增可选 `id` 参数用于精确匹配，按 id > title+time 优先级查找
- 系统提示词新增「上下文关联」章节，明确要求 LLM 将短时间内连续消息关联为同一件事
- 明确禁止因关键字（如"开会"）错更新到其他日程
- 更新通知规则：更新日程时也必须调用 notify_user

## 原因
- 用户说"今天晚上11点钟有个事情"创建日程后，紧接着说"在4教312开会"，LLM 错误地更新到了旧日程"会议"上
- LLM 因为消息含"开会"二字，将地点关联到"会议"而非刚创建的日程
- 事后还编造了"之前记录的地点'六教212'"——实际该旧日程确实存在此地点，但不应被更新

## 变更文件
- src/agent/prompt.rs（新增「上下文关联」章节，更新通知规则）
- src/tools/schedule.rs（schedule_update 新增 id 参数，优先 id 匹配）
