# 修复消息批处理 + 禁止 LLM 编造地点

**日期**: 2026-05-16
**类型**: Bug 修复

## 变更内容
- 新增「消息批处理」规则：短时间内连续到达的多条消息应合并理解为同一件事
- 新增「禁止编造」规则：地点/时间等信息必须逐字使用用户原话
- 移除 prompt 中具体地名例子（如"在4教312开会"），减少对 LLM 的诱导
- 强化「更新日程也必须通知用户」规则

## 原因
- 两条消息同时到达时，LLM 分别创建了两个日程而非合并信息
- LLM 将持续把用户说的"体育馆"编造成之前对话中出现过的"4教312"

## 变更文件
- src/agent/prompt.rs（新增消息批处理/禁止编造规则，移除诱导性地名例子）
