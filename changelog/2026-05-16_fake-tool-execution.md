# 修复 LLM 假执行工具（说已通知但没调工具）

**日期**: 2026-05-16
**类型**: Bug 修复

## 变更内容
- 新增禁止假执行规则：说"已发送通知"但没输出 `<tool_call>` = 没发送
- 工具调用章节加入醒目警告：说要调工具就必须输出对应的 `<tool_call>`
- 加入正确/错误示例对比

## 原因
- LLM 经常输出"✅已通知用户 ✓已调用Claude Code"但完全没输出 tool_call
- 工具不会自动执行，LLM 只是在编造结果

## 变更文件
- src/agent/prompt.rs（工具调用节增加警告，禁止编造节扩展为禁止假执行）
