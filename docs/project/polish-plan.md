# QAgent 完善计划

## 1. 完善的 Web UI

**目标**: 从"能用"到"好用"

- [ ] 暗色主题美化（现在就是 GitHub 暗色风格，但布局简陋）
- [ ] Claude Code 实时进度展示（已存 .claude_progress，面板还没接入）
- [ ] 日程视图增强（日历视图、周视图、过期高亮）
- [ ] 消息历史搜索/过滤
- [ ] 记忆和笔记的管理界面增强（编辑、标签筛选）
- [ ] 移动端适配

## 2. 健壮的代码

**目标**: 跑得稳，挂了能自己恢复

- [ ] LLM 调用失败重试（指数退避）
- [ ] WebSocket 断线自动重连（现在有但可能不够）
- [ ] 上下文超限的优雅降级（现在会崩）
- [ ] 工具执行超时兜底
- [ ] 更完善的日志（结构化、分级）
- [ ] Graceful shutdown（SIGINT 时保存状态）
- [ ] 启动自检（检查模型文件、端口是否可用）
- [ ] **排除列表**：配置忽略的群/用户，不在无关消息上浪费 LLM 上下文

## 3. 高安全性

**目标**: 不泄露、不越权、不被黑

- [x] NapCat token 已从代码和文档清除
- [x] claude_workspace/ 已 gitignore
- [ ] 敏感配置加密或系统密钥链存储
- [ ] Web 面板加简单鉴权（防止局域网其他人访问）
- [ ] 工具沙箱（claude_code 的工作目录隔离）
- [ ] 限制工具访问路径范围（防止读/写敏感文件）
- [ ] 日志脱敏（过滤 token、密码等关键字）

## 4. 智慧的工具

**目标**: 工具更智能、更可靠

- [ ] schedule_update 匹配逻辑加强（模糊匹配、时间邻近匹配）
- [ ] ocr_image 优化（多种引擎降级、缓存）
- [ ] qq_read 扩展（读取更多群信息、历史搜索）
- [ ] 工具执行结果截断策略优化（现在 8192 字节硬截断）
- [ ] 工具调用失败后 LLM 自动调整参数重试
- [ ] 新工具：网页搜索、天气查询、翻译

## 5. 完善的记忆体系

**目标**: 记得住、找得准、不忘记

- [ ] 会话上下文的持久化（现在重启后历史丢失，只有事件记录）
- [ ] 记忆自动老化与摘要（太久远的自动压缩）
- [ ] 跨会话关联（不同群聊提到同一话题时关联）
- [ ] 记忆重要性评分（不重要的自动过期）
- [ ] 主动回忆触发（不是等 LLM 调用 recall，而是系统自动判断需要时注入）
- [ ] embedding 定期批量优化（碎片整理）

## 6. 可观测性

**目标**: 看得见系统在干什么

- [ ] LLM 调用统计面板：token 消耗、响应时间、失败率
- [ ] 工具调用统计：调用次数、成功率、耗时分布
- [ ] 健康检查 API：LLM 状态、WebSocket 连接、内存/CPU
- [ ] Web 面板状态仪表盘

## 7. 一键部署与运行

**目标**: 任何机器上都能一键跑起来

- [ ] 启动脚本：自动检测依赖、启动 llama-server、等待就绪
- [ ] 配置文件化（config.toml 替代杂乱的环境变量）
- [ ] 依赖自动检测：claude、Tesseract、llama-server 是否已安装
- [ ] Docker 化（可选）

## 8. 数据备份与迁移

**目标**: 数据不丢、随时可迁移

- [ ] 自动定时备份 data.db
- [ ] 日程/记忆/笔记导出为 JSON
- [ ] 数据完整性校验（启动时检查）
- [ ] 导入功能（从备份恢复）

## 9. 测试覆盖

**目标**: 改代码不怕改坏

- [ ] 工具单元测试（每个工具独立测输入输出）
- [ ] Agent 循环集成测试（mock LLM，验证工具调用链）
- [ ] WebSocket 重连测试
- [ ] SQLite 存储测试

## 10. 用户自定义配置界面

**目标**: 不写代码就能调
