# QAgent 远景规划

## 核心理念

不只是 QQ 助理，而是一个**统一的信息处理中枢**。接入所有信息来源，统一记忆，自主决策，自动执行。

```
┌─────────────────────────────────────────────────┐
│                 信息接入层                        │
│  QQ · 微信 · 邮件 · Slack · Telegram · SMS · RSS │
└──────────────────────┬──────────────────────────┘
                       ↓
┌──────────────────────┴──────────────────────────┐
│                 统一记忆                         │
│  语义知识库 · 对话历史 · 用户画像 · 事件日志      │
│  向量数据库 + 关系型 + 文件存储                  │
└──────────────────────┬──────────────────────────┘
                       ↓
┌──────────────────────┴──────────────────────────┐
│                 决策引擎                         │
│  LLM 推理 · 优先级判断 · 上下文融合 · 风险检测    │
└──────────────────────┬──────────────────────────┘
                       ↓
┌──────────────────────┴──────────────────────────┐
│                 行动层                           │
│  通知 · 日程 · 自动回复 · 任务执行 · 报告生成     │
│  Claude Code · 代码执行 · API 调用 · 工作流       │
└─────────────────────────────────────────────────┘
```

## 阶段性目标

### Phase 1 — 当前（已完成）
QQ 专属助理，被动响应。

- [x] NapCatQQ WebSocket 监听
- [x] 本地 LLM 推理（Qwen3.5-9B）
- [x] Agent 循环 + 10 工具
- [x] SQLite 持久化 + 语义记忆
- [x] Web 面板
- [x] Claude Code 集成

### Phase 2 — 多源接入
扩展信息源，统一入口。

- [ ] 微信（WeChat / WeCom）
- [ ] 邮件（IMAP / Outlook）
- [ ] 系统通知挂载
- [ ] RSS / 订阅源监控
- [ ] 统一消息格式与路由

### Phase 3 — 统一记忆
跨平台的知识沉淀。

- [ ] 向量数据库（Milvus / Chroma）替代 SQLite embedding
- [ ] 用户画像自动构建
- [ ] 跨会话知识图谱
- [ ] 记忆优先级与遗忘策略
- [ ] 定时自动摘要与归档

### Phase 4 — 自主行动
从被动响应到主动处理。

- [ ] 定时任务引擎（cron 式）
- [ ] 条件触发规则（if-this-then-that）
- [ ] 周期性群聊消息总结
- [ ] 日程到期主动提醒
- [ ] 异常检测与主动通知
- [ ] 多步骤任务编排

### Phase 5 — 智能体网络
多个 Agent 协作。

- [ ] 子 Agent 分工（监控 / 分析 / 执行 / 记忆）
- [ ] Agent 间通信与任务委派
- [ ] 任务队列与负载均衡
- [ ] 自我监控与异常恢复

## 设计原则

1. **本地优先** — 敏感数据不出本地，LLM 本地推理
2. **只读安全** — 默认不发送消息，人工确认后才执行写操作
3. **渐进增强** — 每个阶段向下兼容，可独立启用/禁用
4. **可观测** — 所有决策过程可追溯，Web 面板透明展示
5. **语义驱动** — 不靠规则匹配，靠 LLM 理解意图

## 架构演进

```
现在（QQ 专用）            未来（信息中枢）
┌──────────┐              ┌──────────────────┐
│ NapCatQQ │              │  信息适配器层      │
│          │              │ QQ · 微信 · 邮件   │
└────┬─────┘              └────────┬─────────┘
     ↓                             ↓
┌──────────┐              ┌──────────────────┐
│ Agent    │              │  统一 Agent 引擎   │
│ Loop     │     →        │  记忆 · 决策 · 行动 │
└────┬─────┘              └────────┬─────────┘
     ↓                             ↓
┌──────────┐              ┌──────────────────┐
│ 10 Tools │              │  工具市场 (n 个)   │
└──────────┘              └──────────────────┘
```

最终形态：一个长期运行在后台的**个人 AI 中控**，像人的大脑一样——自动接收信息、关联记忆、判断轻重缓急、主动采取行动。
