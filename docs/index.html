架构

三層智能流水线

消息进入后经过感知、决策、行动三层处理，LLM 在核心循环中自主判断和调度。

QQ 消息 → 感知层 → Agent 循环 → 行动层 → 通知 / 存储
👀

感知层

通过 NapCatQQ WebSocket 实时监听消息，解析文本、图片、文件、@提及等。

  • WebSocket :4447 + HTTP :4444
  • OneBot v11 协议解析
  • @我 / @别人 / @all 三级区分
  • 排除列表跳过无关群/用户
🧠

Agent 决策循环

本地 LLM 分析消息，自主决定调用哪个工具，支持多步推理。

  • Qwen3.5-9B 本地推理（42-48 t/s）
  • 最多 10 轮迭代
  • ToolRegistry 10 个工具
  • 三层记忆：上下文 + SQLite + 工具
  • 语义搜索（embedding 余弦相似度）
⚡

行动层

工具执行结果反馈给 LLM，形成闭环。支持 Windows 通知和外部 CLI 调用。

  • Windows Toast 通知
  • Claude Code 子进程（stream-json）
  • SQLite WAL 持久化
  • Web 面板实时展示
特性

为什么是 QAgent？

不只是一个 QQ 机器人，而是一个真正有理解能力的个人助理。

🔒

纯本地 · 隐私优先

LLM 在你的显卡上运行，数据不出本地。敏感信息不会上传到任何云端。

🔄

Agent 自主循环

收到消息 → LLM 判断 → 调工具 → 看结果 → 继续或完成。不需要预设规则。

🧩

10 个内置工具

通知、日程、笔记、记忆、OCR、Claude Code、QQ 读取等，覆盖日常需求。

💾

三层记忆系统

对话上下文 + SQLite 持久化 + 语义向量搜索，记得住且找得准。

🤖

Claude Code 集成

复杂任务（写文档、编程、分析）自动委托给 Claude Code，实时反馈进度。

📊

Web 监控面板

Axum + SSE 实时展示消息历史、日程、笔记、记忆和工作区文件。
